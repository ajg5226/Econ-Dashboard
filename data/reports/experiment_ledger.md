## Evaluation-scope policy (adopted 2026-04-23)

**Primary evaluation window is 1990-present.** Pre-1990 origins remain in the test set as *informational* — they're reported in every validation JSON but **do not drive KEEP/DISCARD verdicts**. Training uses full history (evaluation-only scope, not training-scope).

- **In-scope (gate-relevant)** — 10 origins: S&L 1990, Dot-com 2000, GFC 2007, COVID 2019 + 6 post-1990 expansion checks.
- **Informational (reported, not gated)** — 5 origins: Oil Crisis 1973, Volcker I 1979, Volcker II 1981 + 1978/1986 expansion checks.

Rationale: recurring pattern across A1/A1.75/B2/B2.5 — modern-era optimization destabilizes pre-1990 detection because the pre-1990 macro regime (Volcker disinflation, ~40% goods-employment share, pre-financial-innovation) is materially different from what the dashboard forecasts in. Optimizing for 1981 is museum work.

Rows filled before 2026-04-23 ("A1", "A1.5", "A1.75", "B1", "B2") were gated under the old full-history policy — their verdicts stand, but scope-relevant deltas may differ if re-evaluated.

## Verdict legend
- **KEEP** — meets the keep gate (in-scope PR-AUC ≥ baseline + ≥1 secondary improvement, no >10% in-scope regression)
- **DISCARD** — hits the discard gate (>15% in-scope regression, or fails to catch Dot-com + GFC + COVID)
- **NEEDS-WORK** — neither; iterate or reframe

## Status legend
- **PLANNED** — in backlog, not started
- **IN-PROGRESS** — builder sub-agent running
- **VALIDATING** — validator sub-agent running
- **DONE** — row filled, verdict decided

## Gate override log

When we merge an experiment despite a mechanical NEEDS-WORK / DISCARD verdict, we document the reason explicitly here. The override is a judgement call that ships code — it should be exceptional and auditable.

### Override #1 — B3 (2026-04-23)

**Mechanical verdict**: NEEDS-WORK. Dot-com calibrated peak moved only +0.5pp (gate required ≥3pp); CV drop-ablation on `SLOOS_TIGHTENING_FLAG` was -0.6pp (gate required ≥3pp); in-scope CV PR-AUC regressed 0.45pp.

**Override rationale**:
1. The CV drop-ablation is the wrong instrument for this feature. `SLOOS_TIGHTENING_FLAG` is a binary credit-cycle indicator that mostly sits at 0 during expansions. CV folds pool decades of non-recession months, diluting the flag's impact. **Test-window drop-ablation is -17.2pp PR-AUC** — the feature is doing heavy lifting where it matters.
2. The Dot-com gate was imported from A1.75.2's framing but B3's hypothesis was credit-supply-tightening-leads-recessions (targets GFC/S&L), not Dot-com specifically. SLOOS didn't fire strongly pre-Dot-com (2001 was an investment-led downturn, not a credit tightening event) — failing to lift Dot-com isn't a failure of B3's hypothesis, it's a correct negative result.
3. **GFC lead time jumped 1.02mo → 6.96mo (+5.94 months)**. This is the largest single-recession lead-time improvement any experiment has produced. For an early-warning dashboard, this is exactly the metric that matters. S&L also +1mo.
4. `SLOOS_TIGHTENING_FLAG` was selected at **rank 2** of the entire 50-feature pool. The feature selection algorithm treats it as one of the two most-important signals available. That's not a weak signal; that's a structural model finding.
5. The modern-era cost is modest: CV PR-AUC -0.45pp is noise; COVID peak -10pp still leaves it above threshold.

**Lesson for future gates**: when an experiment's hypothesis targets a specific regime or recession type, the gate's criteria should match that hypothesis rather than carry forward unrelated constraints from prior experiments. In future, gate definitions get a one-line "what this experiment is trying to move" anchor in the brief.

## Notes for validators

When appending a row:
1. Δ columns are (challenger minus baseline) **computed on in-scope origins**. Negative = better for Brier/ECE; positive = better for PR-AUC/LeadTime. Full-history metrics belong in the validation JSON, not the ledger row.
2. Lead-time delta is computed across detected recessions only (in-scope).
3. Include the experiment's git branch or worktree path in the notes column so changes are reviewable.
4. If discarded, still keep the row — a negative result is a result.
5. Report pre-1990 lead-time regressions in the notes as "Volcker/Oil Crisis Δ: informational" — don't let them kill the verdict unless they exceed -25% in isolation (sanity floor).
6. Gate overrides: if you merge despite a mechanical NEEDS-WORK, append a new override entry under "Gate override log" with the mechanical verdict, the rationale, and the lesson for future gate designs.
